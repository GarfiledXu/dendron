## c

1. 指针是什么？和指针相关的操作有哪些？指针的类型信息有哪些作用？这些类型信息主要是提供给编译器识别还是在程序运行时？void类型指针相比有类型指针，哪些操作不能执行?
   1. 指针本质上是一个 保存内存地址，并且带有类型信息的特定类型变量，更完整的称呼应该是指针变量
   2. 操作:
      1. 取地址(&a)
      2. 解引用(*p)
      3. 指针算数(++, +1)
   3. 指针的类型信息非常关键，并且这是属于语法层面的静态属性，仅在编译期指示编译器进行机器指令的生成，决定了运行时，cpu对具体内存的操作方式。即编译器在编译期就将指针的类型信息 转化固化为了运行时执行的机器指令
      1. 决定了解引用时的步长(stride):即确定了地址以后读取多少长度的字节
      2. 内存二进制数据的解释方式:同样长度的数据是按整数还是浮点数解释
      3. 以及指针算术运算后的偏移量offset: 如int类型的指针*p++后，地址偏移应该是sizeof(int)=4，而不是1
   4. void类型指针
      1. 不能直接解引用，如：*p，必须经强转，如：*(int*)void，即先转化为有类型指针，再解引用
      2. 不能执行算术运算，如自增或者+1
   5. 对于所有指针：指针变量的相加以及乘法，除法都没有什么意义
   6. 在同一个ABI架构中，所有类型的指针，其对象占用的空间大小都是固定的，经常提到的概念操作系统位数通常指的就是系统ABI地址宽度，而指针本身就是虚拟地址，所以与ABI宽度相等
2. static 关键字的作用？
   1. 在c语言中
      1. 修改变量的生命周期: 如修饰局部变量时，静态局部变量 则存储在 .data或者.bss 段，生命周期伴随着整个程序，而不是存储在栈中
      2. 修改符号的链接属性: 如修饰全局变量时，静态全局变量，只能同一个文件编译单元可见，是内部链接
      3. 通常 函数存储在text段，全局变量，静态局部变量存储在.data .rodata .bss段，而字符串字面量和const变量存储在 .rodata，注意! `整数值字面量存储在.text段中和指令一起存储即立即数，浮点数值字面量存储在.rodata中`
   2. 在c++中
      1. c++多了，static可以修饰类成员和成员函数
   3. 初始化顺序的差异 关键，这是初始化顺序，不是存储顺序
      1. 首先初始化的方式，c语言静态变量初始化只能赋值 编译常量，是不能函数动态初始化的，而c++是可以的
      2. 其次初始化的顺序，c语言不管全局还是局部的静态变量都是在main函数以前，完成初始化，并且只能编译常量初始化。而c++要进行分类，全局，以及类静态成员变量都是在main函数执行之前，并且可以执行函数，动态初始化，而局部静态函数则是运行到对应语句时，才执行初始化，这也是很多c++框架，会用来实现静态加载的机制
3. sizeof是什么？是函数还是运算符?对输入对象的类型是否有要求?返回的数据类型和单位是什么?执行时机是在运行时处理还是编译期处理?有哪些经典问题?c++中的空类，和c语言中的空结构体，sizeof是否有区别，大小是多少?sizeof的计算方式以及对表达式的处理?为什么sizeof对输入表达式的处理叫做 解析而不是执行?
   1. sizeof是编译期运算符，不是函数，目的就是计算输入符号的类型大小，不求值，返回size_t类型数据，单位是字节，即表示类型大小
   2. 合法的输入形似
      1. 直接具体的类型名 sizeof(int)
      2. 变量名称 sizeof(aaa)
      3. 输入表达式也可以 `sizeof(a + b) sizeof(28*35) sizeof(a*5*b+a)`
   3. 经典案例1: `int* p; sizeof(*p)=4;` 最终编译期sizeof是如何得到int的?为何不叫表达式执行?
      1. 通过语法分析，判断得到这个表达式的最终类型为解引用后对应的类型，即int
      2. 进行语法分析不等同于执行，在虚拟计算机视角的执行对应到编译期和cpu运行时的行为，本质上应该是 编译期在编译期生成执行指令，然后运行时cpu执行，因此关键在于 表达式解析后 编译期有没有为其生成执行指令的行为，有则是执行，否则本质上sizeof这里仅仅是进行了语法解析，目的是提取出最终的类型
   4. 经典案例2: `int a,b,c; double d; sizeof(a*b+c*d-d)` 这里会对表达式逐层解析，每一步都会合并得到一个类型，类型大的会吞并类型小的，即类型提升，最终确定为double类型，即8
   5. 特殊规则:整形提升规则:所有比 int 小的整数类型，在参与表达式运算时，都会先提升为 int（或 unsigned int）
      1. 即只要组成表达式了，那么这个表达式的类型值只会比int大，`char a; short b;sizeof(a+b)` 结果是int
      2. `short a; short b; auto x = a + b;` 那么最终auto推导出来的类型应该为int
   6. c99的特殊:可变数组，故c99的sizeof有可能是运行时计算的，如果是对可变数组进行判断类型大小
   7. 空类在c和c++中的差异，在c语言中允许空类为大小空，那么地址也可以相同了，而c++空类必须要有大小，且地址唯一，也就意味着相同代码，sizeof结果是不同的
   8. 数组类型和函数传参的特殊性，即数组退化
      1. 函数参数为数组类型时，进入到函数内部，类型会退化为指针类型，即sizeof固定大小为8字节
      2. 原因：主要是因为用转化为指针后传递效率高，否则直接通过数组值拷贝，会由于数组长度而爆栈

4. extern关键字的作用？ 核心：不分配空间，是声明，不是定义
   1. extern的关键字是用来 声明所修饰的符号 `是外部符号`，因此，编译器不会给该变量分配空间，只会标识该符号为`undefined symbol`用于后续链接的时候，从其他目标文件中查找

   5. 问题：extern 可以修饰函数吗？答案: 函数默认就是extern

13. 由浅入深的指针类型？
    1. 指针解引用的定义和语义
       1. 可以看作是一个运算过程: 输入特定类型的表达式，进行`求值`（而不是取址，因为这是一个运算过程），获取地址值，然后进行对象`指代 denote`（而不是对象构造，创造），返回目标左值表达式，用于具体上下文使用
       2. 解引用的操作对象：输入 指针类型的表达式（既可以是指针变量，也可以是地址常量，关键是指针类型）
       3. 解引用的结果：输出/产生 左值表达式（该表达式指代具体的对象，而不是`值容器`，只有到具体上下文中，才会触发对应的对象和值转换）
    2. 指向变量的指针，`int* p, q;  // ❌` p是int*，q是int，指针的*是和变量名绑定的，不是和类型绑定的
    3. 多级指针，`int a = 10; int *p = &a; int **pp = &p;` 关系链：pp → p → a，pp 是一个 指向 int 指针的指针
    4. const和指针
       1. `const int *p; int const *p; int * const p;`
          1. 通过指针，不能改什么，可以改什么？
             1. *p = 10;   // ❌;p = &b;    // ✅
             2. *p = 10;   // ✅;p = &b;    // ❌
    5. 数组和指针
       1. `int arr[5]; int *p = arr;`
          1. arr 在表达式中会退化为 int*
          2. p[i] ≡ *(p + i)
          3. 数组访问本质上就是指针运算
       2. `int arr[5];int (*p)[5] = &arr;`
          1. p 指向的是 整个数组
          2. p + 1 跳过 5 个 int
          3. 指针步长由“它指向的类型”决定。
    6. 函数和指针
       1. `int add(int a, int b);int (*fp)(int, int) = add;`
          1. fp 保存的是函数入口地址
          2. fp 保存的是函数入口地址
          3. 函数名在使用时会退化为函数指针
    7. 函数+数组+指针
       1. `int (*ops[3])(int);`
          1. ops[3] → 数组
          2. 元素是指针
          3. 指向 int(int) 函数
          4. 应用：ops[0] = open;ops[1] = read;ops[2] = close;
    8. 函数返回+指针
       1. `int *get_buf(void);`
       2. 对比：`int (*get_cb(void))(int);`
       3. 返回函数指针（回调注册）
14. 指针传参的思考，什么时候使用一级指针，什么时候使用二级指针？
    1. 准确来说是一级指针参数和二级指针参数，因为我们讨论的上下文就是函数的参数传参
    2. 函数的指针传参，本质上就是定义一个临时指针变量进行值拷贝
    3. 一级指针参数解引用后，可以获取一级指针指代的对象，执行对象相关操作，但不能修改指针本身的指向，因为函数指针传参，是临时变量拷贝，而不是引用
    4. 二级指针参数解引用后，可以获取二级指针指代的一级指针对象，执行一级指针的相关操作，即使二级指针本身是一个临时变量拷贝，也能完整对一级指针的修改
    5. 所以取决于函数是否需要修改“目标指针对象（注意，当发生了临时变量的值拷贝，那么二者存储了相同的地址值，但分属于两个对象）”，比如对目标指针对象进行内存的分配、指向新对象、希望释放指针内存的同时对指针赋值置空、链表等数据结构的操作（比如需要表头插入一个节点，那么就需要传入待插入节点的一级指针，和当前链表的二级指针，）

15. 运算符分类，优先级 TODO
    1. 按目进行分类，则分为`单目` `双目` `三目`，何为目即"操作数"
    2. 常见示例：
       1. 括号和成员访问永远最优先
          1. a.b.c 到底是 `(a.b).c` 还是 `a.(b.c)`? 答案是结合应该从左到右，那么就是`(a.b).c`
          2. p->a[3] 到底是 `p->(a[3])` 还是 `(p->a)[3]`？答案是 `(p->a)[3]`

          1. *p++？ 实际上是`*(p++)`
       3. 算术运算内部的顺序，`* / %` 优于 `+ -`
       4. 算术，比较，移位，逻辑
          1. a << 1 + 2? 等同于 `a << (1+2)` 加减是要优先移位的，即算术优先移位
          2. a > b && c > d 等同于 `(a > b) && (c > d)` a == b || c == d 等同于 `(a == b) || (c == d)` 即比较也是优先逻辑的，但是移位是优于比较的，记忆（移位移位比比较是double）

16. 解释一下，什么是内存对齐?谁为谁，做了什么?内存对齐时字节填充的规则?
    1. 在编译期，编译器为了cpu对内存读取的效率，会给一个对象填充字节，比如给结构体的成员间填充字节
    2. 字节填充的规则: 每个成员对地址有要求，要求地址是其自身大小的整数倍，也就是要先选址，选好以后，前面空闲的内存就填充
       1. 起始地址偏移对齐，每个结构体成员，先判断当前地址偏移是否是自己的大小的整数倍，不是则往后挪挪，前面空闲出来的则填充，所以大小为1的不需要考虑对齐，因为肯定对齐，例如: 1，4，1的顺序和大小结构体，对齐后 1，+3，4，1即（4，4，1）
       2. 收尾对齐，整体大小规则(必须是最大成员的整数倍)，故整体数据大小要12，那么最后一个字节放置后还需要填充3字节。故定义时成员的排列顺序会影响内存对齐的占用大小，最优规则就是按成员大小从大到小排列，这样可以保证后续成员的因为偏移需要填充的字节最少.
       3. 对齐参考的单位即结构体中最大成员的大小.

## 逻辑概念

1. 概念 concept
   1. 对对象简化模糊的描述，用于指代，讨论，交流
2. 定义与语义
   1. 定义 definition：作为名词时，是一个陈述描述，用明确的实体和基本属性来规范概念，设置判断标准，从事物的表象和属性着手
   2. 语义 semantics：作为名称时，是一个陈述描述，用抽象的方式解释具体语法的行为和效果，描述通常无关底层实现，而是注重抽象逻辑
   3. 移动语义，求值语义，行为语义，上下文语义，这些都是具体且不同维度上的解释规则集合
   4. 编程语言中语义的核心用途：
      1. 约束和指导语言实现（编译器/解释器），语义是编程语言抽象的规则并不是底层实现，但是语言的`实现必须服从语义`
      2. 给编程人员提供理解和推理程序行为的抽象依据
3. 行为和效果
   1. 在日常口语化的表达中，二者经常被混用，互为替代
   2. 更准确的定义：行为是因，就是描述发生的动作和过程，而效果是果，描述最终的结果，状态变化
   3. 为何刻意区分：在语言标准或者语义中，有些行为不一定有可观察结果，有些效果也不一定能够暴露行为过程，举例：
4. 表达式 expression
   1. 本质上是语法结构，一种能够被求值，产生结果的语法结构，这个结果可能是`值`，也可能是`抽象的实体对象`
5. 左值和左值表达式 lvalue
   1. 左值，首先是被口语简化的概念，准确描述就是`左值表达式`，字面上仅是表达式的一种类别属性，指代`表达式的一个类别`
6. 对象 object
   1. 计算机语言中抽象出来的实体概念，在`程序的执行过程`中被创建，且具有确定的存储区域和生命周期的`实体`
7. 变量 variable
   1. 是一种语法层面的标识符，变量不是对象的具象化，而是对象的一种`命名方式`，是对具体对象的`命名绑定`，即`有名称的对象`
8. 左值表达式和对象和变量
   1. 变量名本身可以是表达式或者其组成，表达式本身在语义上也可以指代某个对象（比如通过指针运算）
   2. 对象属于程序的抽象实体，而变量则是语法标识符，是对对象的命名绑定
9. c/c++语义 对象 地址 指针 内存分配 内存释放
    1. 在c/c++的语境中讨论内存以及地址等相关概念，最核心的就是对象
    2. 对象是在语言层面抽象出来的实体概念，是具有类型，存储和生命周期规则的抽象实体，其他概念均为其服务
    3. 变量是对象在语法层面的标识符
    4. 地址是对象存储位置的标识值
    5. 指针是一种类型的对象，该对象存储的值是地址值
    6. 所以对象和地址的关系，不是唯一映射关系，更准确的说法：在对象的生命周期内，对象与地址的关系是唯一映射的
    7. 内存的分配和释放malloc和free
       1. 在语义层面上，free的输入严格要求了就是由 malloc / calloc / realloc 返回的，且有效分配的地址值
       2. 在语法层面上，输入free的表达式能够进行求值，返回地址值就是合法的
       3. malloc则是生成一个新的可用的分配了内存的对象，并返回其对应的地址值
       4. 故：通过一级指针参数无法修改指针指向其他对象或者再分配内存，因为你malloc的地址只能赋值给临时变量（指针对象），而不是原指针对象
       5. 故：通过一级指针参数可以free对应的内存，但是无法将原指针对象置NULL，只有二级指针参数才能解引用后，获取到指向原指针对象，修改其存储的地址值
10. 抽象机器 表达式的 求值 取值 取址 编译时 运行时
    1. 何为抽象机器（abstract machine）：
       1. 在c/c++标准中有提及该概念，原文`The semantic descriptions in this International Standard describe the behavior of an abstract machine in which issues of optimization are irrelevant.` 本标准对程序语义的描述，是基于一个“抽象机器”，在这个机器里，不考虑优化和具体实现细节
       2. 本质上是一个`假想的执行模型`，用于语言标准 `定义和讨论程序语义`，强调是为了定义语义，而不是用于指导实现
    2. 求值 取值 取址 讨论的上下文：基于抽象机器的执行模型，不区分编译期和运行时
    3. 三种概念并不是语法规则，都是在抽象机器的语义上下文中，表达式语义的几种行为过程，求值是过程统称，取值和取址都是特定上下文下的语义行为
    4. 求值 evaluation：抽象机器对表达式进行语义处理并确定结果的`过程`，结果可能是一个值，也可能是一个对象的指代
    5. 取值 value computation/read：抽象机器在求值表达式时，会根据表达式所处的语义上下文判断是否需要对象的值；若需要，则读取该对象的存储内容的`过程`
    6. 取址 address computation：抽象机器在求值表达式时，当语义规则要求产生对象的地址值时，对该对象执行地址计算并生成指针值的语义行为

        ```c++
         int z = *p + 1;    // 求值：二元表达式
                     //
                     // *p：
                     //   - 求值为对象指代（x）
                     //   - 上下文需要值
                     //   - 取值：读取 x 的内容
                     //
                     // + 1：
                     //   - 对值进行计算
        ```

    求值

在抽象机器的语义上下文中，对表达式进行语义处理以确定其结果的过程；结果可能是一个值、一个对象的指代，或仅产生副作用。

✔ 取值

当表达式求值的结果是对象指代，且上下文需要其值时，抽象机器对该对象存储内容进行读取的语义行为。

✔ 取址

对一个指代对象的左值表达式应用取地址运算，构造一个指向该对象的指针值的语义行为。
