# Debug 客户端初期需求文档

## 1. 项目定位与目标
本工具基于 DearPyGui (DPG) 构建，定位为视觉底层框架（C/C++）的**专属开发者白盒测试工作站**。
它不用于最终商业交付（商业交付由 Qt/QML 负责），其核心目的是在底层引擎研发阶段，提供极速的 DAG 拓扑验证、通信协议防呆测试、内存状态透视以及偶发 Bug 的现场还原能力，极大提升上下位机联调效率。

## 2. 核心架构与技术栈
* **前端渲染**：DearPyGui (保留模式 API，底层 GPU 立即模式直绘)。
* **数据总线**：Python `ctypes` / 共享内存 (处理高频 YUV/RGB 图像裸指针) + JSON (处理控制信令和阈值参数)。
* **图像与矩阵**：NumPy + OpenCV (`cv2`)。
* **UI 布局**：Docking 停靠布局，支持多面板自由拖拽、平铺与折叠。

## 3. 核心功能模块划分

### 3.1 视觉交互与拓扑编辑面板 (Vision & DAG Board)
作为最基础的操作沙盒，提供核心的视觉与逻辑编排能力。
* **图像渲染窗**：支持接收底层传来的 YUV/JPG/内存裸流，并实现 60FPS 的 GPU 硬件加速预览。
* **ROI 交互层**：支持鼠标平滑框选、拖拽几何体（矩形、多边形等）。
* **DAG 节点编辑器**：内置基于 DPG Node Editor 的拓扑编辑器，支持添加算子节点（Atool）、逻辑节点（Logic），并自由连线以生成依赖关系图。

### 3.2 报文双向映射与注入控制台 (Bi-directional Payload Mapper)
替代传统的干瘪输入框，实现“所见即所得”与“代码即 UI”的双向同步，是协议联调的核心测试场。
* **正向生成（UI -> Payload）**：
  * 用户在图像上画一个框，或者在节点上拖动一个滑块。
  * 面板实时生成对应的 JSON 配置报文（如 `{"x": 10, "y": 20, "width": 100, "height": 100}`），并高亮显示变动字段。
* **逆向解析（Payload -> UI）**：
  * 开发者在面板的文本框中手动输入或粘贴一段 JSON 报文。
  * 系统解析 JSON，反向驱动图像窗口在指定坐标自动生成高亮的 ROI 框，或自动修改对应节点的 UI 状态。
* **极限注入测试**：允许开发者故意修改 JSON 字段（如非法参数、越界数值），一键注入给 C++ 底层，测试引擎的防呆与容错能力。

### 3.3 核心状态白盒透视镜 (Core State Inspector)
用于实时监控下位机 C++ 内存中不可见的流转数据。
* **Pipeline Frame 监控树**：树状结构展示当前黑板（`prog->pipeline_frame`）中所有存活的 Key 与 Value 类型。
* **实时刷新与高亮**：每次 `run_once` 执行后，自动刷新列表。对本次流转中发生变更的数值、新产生的结果图进行闪烁高亮。
* **指针内容可视化**：点击任意图像指针 Key，侧边栏直接渲染该中间态图像；点击数据结构指针，展开显示详细数值。

### 3.4 虚拟硬件 I/O 仿真器 (Virtual Hardware I/O)
闭环测试视觉输出与外部物理设备的握手逻辑。
* **输入仿真 (IN)**：提供软件触发按钮（模拟光电开关）、外部信号输入切换，支持设定 `inner_delay` 和 `ext_delay`。
* **输出仿真 (OUT)**：提供 1~N 个虚拟指示灯（对应物理 Port）。接收底层 `output_res_node` 的结果映射，实时亮起红/绿灯。

### 3.5 算子性能火焰图 (Performance Profiler)
监控底层在不同工作模式（Standard / Division / Cross）下的实时性能。
* **帧率与耗时监控**：使用 ImPlot 绘制滚动折线图，显示总体调度帧率。
* **节点级耗时拆解**：以柱状图或甘特图形式，展示单次 `run_once` 内部，各个节点（`PRE_CSC`, `Atool_N`, `TOT_RES`）的精确毫秒级耗时，快速定位算子瓶颈。

### 3.6 决定性录制与回放系统 (Deterministic Record & Replay System)
**本工具的核心杀手锏。** 用于隔离偶发环境因素，实现 Bug 的 100% 完美复现与对齐。
* **多维度时间轴录制**：
  * **输入流录制**：记录每一帧的原始图像数据（或图像缓存指针）及硬件触发时间戳。
  * **操作流录制**：记录用户在上位机的所有 UI 动作（连线、改参、拖框）。
  * **日志流录制**：同步抓取底层打印的各类 Log。
* **精细化回放控制**：
  * 支持以 1x, 0.5x 等速度播放，或**单步步进（Step-by-Step）**。
  * 在回放时，双向映射器、状态透视镜、性能监控面板将完全按照录制时的数据同步重绘，实现真正的“时光倒流”。
  * **核心价值**：当发现算法漏检或内存崩溃时，直接保存该次回放文件包（Trace Package），彻底消灭“在我的电脑上跑是正常的”这种沟通推诿。

### 3.7 沉浸式高级日志台 (Advanced Log Console)
* **功能**：接收 C/C++ 抛出的结构化日志，摒弃传统终端。
* **特性**：
  * 支持按模块（如 `np_mod`, `npm_mod`）、按级别（DEBUG, WARN, ERROR）进行多维度过滤。
  * 关键字正则表达式高亮。
  * 日志条目支持点击追溯（联动回放系统，点击某条 ERROR，回放时间轴自动跳转到对应的那一帧）。

## 4. 后续规划
本工具将伴随底层框架的研发全程演进。在其内部沉淀出的高质量 JSON 交互协议、容错策略以及稳定的 ctypes 跨语言通信封装，将直接作为最终商业版上位机的核心中间件代码，实现研发资产的 100% 复用。
