12. 由浅入深的指针类型？
    1. 指向变量的指针，`int* p, q;  // ❌` p是int*，q是int，指针的*是和变量名绑定的，不是和类型绑定的
    2. 多级指针，`int a = 10; int *p = &a; int **pp = &p;` 关系链：pp → p → a，pp 是一个 指向 int 指针的指针
    3. const和指针
       1. `const int *p; int const *p; int * const p;`
          1. 通过指针，不能改什么，可以改什么？
             1. *p = 10;   // ❌;p = &b;    // ✅
             2. *p = 10;   // ✅;p = &b;    // ❌
    4. 数组和指针
       1. `int arr[5]; int *p = arr;`
          1. arr 在表达式中会退化为 int*
          2. p[i] ≡ *(p + i)
          3. 数组访问本质上就是指针运算
       2. `int arr[5];int (*p)[5] = &arr;`
          1. p 指向的是 整个数组
          2. p + 1 跳过 5 个 int
          3. 指针步长由“它指向的类型”决定。
    5. 函数和指针
       1. `int add(int a, int b);int (*fp)(int, int) = add;`
          1. fp 保存的是函数入口地址
          2. fp 保存的是函数入口地址
          3. 函数名在使用时会退化为函数指针
    6. 函数+数组+指针
       1. `int (*ops[3])(int);`
          1. ops[3] → 数组
          2. 元素是指针
          3. 指向 int(int) 函数
          4. 应用：ops[0] = open;ops[1] = read;ops[2] = close;
    7. 函数返回+指针
       1. `int *get_buf(void);`
       2. 对比：`int (*get_cb(void))(int);`
       3. 返回函数指针（回调注册）
13. 指针传参的思考，什么时候使用一级指针，什么时候使用二级指针？
    1. 因为函数的指针传参，本质上定义一个临时指针变量进行拷贝，
    2. 取决于函数是否需要修改“指针本身”，比如进行内存的分配、指向新对象、希望释放指针内存的同时对指针赋值置空、链表等数据结构的操作（比如需要表头插入一个节点，那么就需要传入待插入节点的一级指针，和当前链表的二级指针，）
    3. 一级指针=用东西；二级指针=换东西
