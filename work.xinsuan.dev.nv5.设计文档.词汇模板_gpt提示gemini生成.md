# 工业级架构设计文档专属词汇库与句式体系 (Industrial Architecture Vocabulary & Templates)

## 1. 标题命名公式与名词后缀矩阵

**通用命名公式：**
`[作用域/模块边界] + [核心实体/操作/数据结构] + [工程名词后缀]`

### 1.1 核心工程名词后缀 (Suffix Matrix)

此类后缀用于精准界定文档章节的工程性质，彻底替代“说明”、“处理”、“抽象”等泛化词汇。

* **规约 (Specification / Convention)**：多方强制遵守的格式或行为契约（例如通信、ABI）。
  * *替代*：“交互说明”、“协议契约”。
* **约束 (Constraint)**：系统、硬件或逻辑上的物理/非物理限制。
  * *替代*：“注意事项”、“开发要求”。
* **机制 (Mechanism)**：维持系统某项功能运转的内部逻辑闭环（含触发、流转、反馈）。
  * *替代*：“处理流程”、“实现原理”。
* **策略 (Strategy / Policy)**：在受限资源下，应对特定场景的算法或行为选择倾向。
  * *替代*：“解决办法”、“优化方案”。
* **拓扑 (Topology)**：物理实体、进程、网络或线程间的空间/逻辑连接关系。
  * *替代*：“系统架构图说明”、“连接方式”。
* **模型 (Model)**：剥离业务细节后，对并发、内存、状态机等底层特征的泛化概括。
  * *替代*：“业务抽象”、“设计思路”。
* **路径 (Path)**：控制流或数据流在系统内部的执行轨迹（如热路径、冷路径）。
  * *替代*：“执行流”、“代码走向”。
* **布局 (Layout)**：内存、磁盘、缓存或结构体在物理/虚拟地址空间的字节级排布。
  * *替代*：“内存分配图”、“结构说明”。
* **通路 (Pipeline / Channel)**：数据跨越边界（如软硬件边界、进程边界）的高速传输通道。
  * *替代*：“数据传输方式”。
* **阶段 (Phase)**：生命周期或复杂状态机中的离散时间切片。
  * *替代*：“执行步骤”。
* **语义 (Semantics)**：指令、状态码或特定操作在当前上下文中的确切物理/业务含义。
  * *替代*：“字段含义”。
* **编排 (Orchestration)**：对多个异步实体、多线程或多硬件组件的时序调度控制。
  * *替代*：“调用顺序”。

### 1.2 工业级标题命名速查表

* **协议/数据流类**：`上下位机同步指令帧拓扑与报文规约` / `DMA 图像数据零拷贝通路设计` / `分片传输重组与状态收敛机制` / `非对齐负载解包路径`
* **内存/结构体类**：`跨平台 C/C++ 结构体 ABI 对齐约束` / `算法句柄内部执行上下文内存布局` / `大页内存映射与防碎片策略`
* **生命周期类**：`算法上下文初始化 Mode 隔离与生命周期闭环` / `IPC 共享内存挂载与卸载阶段定义` / `资源归属转移路径`
* **并发/调度类**：`多相机并发触发时序编排约束` / `高频采集中断服务 (ISR) 抢占与避让策略` / `无锁任务队列生产消费模型`
* **状态机类**：`主控状态机单向流转规约` / `异常挂起与状态机重置路径` / `单一状态源 (SSOT) 维护机制`
* **配置/存储类**：`参数持久化落地与原子刷盘机制` / `运行时配置热重载内存双缓冲模型` / `文件映射 (mmap) 边界约束`
* **容错/异常类**：`非可信指令解析前置防御机制` / `执行域超时挂起与资源强制回收策略` / `协议层 CRC 校验与降级重传通路`

## 2. 工程化修饰定语（前缀）词库

在核心名词前叠加以下定语，可迅速赋予文本强烈的资源边界感、状态约束感与系统底层语境。

* **内存与资源布局 (Memory & Resource)**
  * 预分配的 (Pre-allocated) / 预留的 (Reserved)
  * 零拷贝的 (Zero-copy) / 旁路的 (Bypass)
  * 对齐的 (Aligned) / 紧凑的 (Packed)
  * 连续的 (Contiguous) / 离散的 (Fragmented)
  * 独占的 (Exclusive) / 共享的 (Shared)
  * 池化的 (Pooled) / 多路复用的 (Multiplexed)
  * 易失的 (Volatile) / 常驻的 (Resident) / 脏的 (Dirty)
* **并发与线程模型 (Concurrency & Threading)**
  * 无锁的 (Lock-free) / 互斥的 (Mutex-guarded)
  * 线程安全的 (Thread-safe) / 非线程安全的 (Thread-unsafe)
  * 可重入的 (Reentrant) / 不可重入的 (Non-reentrant)
  * 原子性的 (Atomic) / 分裂的 (Torn)
  * 抢占式的 (Preemptive) / 协作式的 (Cooperative)
  * 阻塞的 (Blocking) / 非阻塞的 (Non-blocking) / 异步挂起的 (Async-suspended)
* **时序与状态调度 (Timing & State)**
  * 单调的 (Monotonic) / 确定的 (Deterministic)
  * 瞬态的 (Transient) / 稳态的 (Steady-state)
  * 严格同步的 (Strictly-synchronous)
  * 延迟绑定的 (Late-binding) / 热重载的 (Hot-reloaded)
  * 上下文感知的 (Context-aware)
* **容错与异常行为 (Fault-tolerance & Exception)**
  * 防御性的 (Defensive)
  * 幂等的 (Idempotent)
  * 容错的 (Fault-tolerant) / 降级的 (Degraded) / 兜底的 (Fallback)
  * 自恢复的 (Self-healing) / 隔离的 (Isolated)
  * 非可信的 (Untrusted) / 被污染的 (Tainted)
  * 静默的 (Silent, 如静默失败) / 显式的 (Explicit)

## 3. 高频工业级术语与表达矩阵

直接用于替换泛泛而谈的口语化描述，精准击中系统设计的痛点区域。

### 3.1 资源与边界管控

* **资源归属 (Resource Ownership)**：界定指针或内存块由谁负责释放。
* **生命周期闭环 (Lifecycle Closed-loop)**：确保 `Alloc` 与 `Free`，`Init` 与 `Deinit` 的必然配对。
* **伪共享 (False Sharing) / 缓存行失效 (Cache Line Invalidation)**：多线程在同一缓存行操作不同变量导致的性能抖动。
* **内存碎片化 (Memory Fragmentation)**：长期运行导致的内存块不连续。
* **上下文逃逸 (Context Escape)**：局部变量或内部指针被非法传递至生命周期外部。
* **资源饥饿 (Resource Starvation)**：低优先级任务长期无法获取执行权。

### 3.2 状态机与时序控制

* **单一状态源 (Single Source of Truth, SSOT)**：系统中的某个状态只由唯一的模块维护，禁止多头写入。
* **状态漂移 (State Drift)**：内存状态与物理设备实际状态产生不一致。
* **状态收敛 (State Convergence)**：从多个并发分支或异常分支最终汇聚到一个确定的稳态。
* **时序敏感区 (Timing-sensitive Zone)**：对中断响应延迟或执行耗时有严格要求（如微秒级）的代码段。
* **竞争窗口 (Race Window)**：锁释放与下次加锁之间，可能被其他线程插入执行的时间间隙。
* **执行悬挂 (Execution Hanging)**：等待永远不会到来的信号量或硬件中断，导致线程假死。

### 3.3 协议层与跨环境约束

* **ABI 漂移 (ABI Drift)**：因编译器版本或编译选项不同导致的结构体大小/虚函数表差异。
* **数据对齐惩罚 (Data Alignment Penalty)**：ARM 等架构下访问非对齐内存导致的异常或性能下降。
* **透明荷载 (Transparent Payload) / 不透明荷载 (Opaque Payload)**：传输层不关心/不可见的上层业务数据区。
* **大端/小端序规约 (Endianness Convention)**：跨平台通信时的多字节解析基准。

### 3.4 异常路径与防御

* **行为退化 (Behavioral Degradation)**：遇到资源瓶颈或硬件异常时，系统放弃部分功能以保证主链路存活。
* **毒药负载 (Poison Pill)**：可导致解析器崩溃或无限分配内存的恶意/损坏数据包。
* **脏数据传播 (Dirty Data Propagation)**：未经验证的错误输入进入内部核心逻辑，导致大面积状态污染。
* **幽灵状态 (Phantom State)**：硬件已复位但软件层仍保留的历史遗留状态。

## 4. 万能句式模板库 (Paragraph Templates)

以下模板可直接用于文档相应章节的段落起始，确立工业级设计的基调。

### A. 模块职责与边界 (Module Boundaries)

* **定义状态归属**：“本模块对 `[底层硬件句柄/图像队列]` 具备唯一的资源所有权（Ownership），负责维护 `[采集状态]` 的单一状态源。外部模块如需获取状态，一律通过只读接口查询，严禁跨边界越权写入。”
* **界定职责排除**：“本层的核心职责收敛于 `[协议反序列化与路由分发]`，完全剥离具体的 `[算法测算逻辑]`。任何与业务强相关的运算堆栈必须下沉至 `[算法核心库]` 执行，以确保通信层上下文的纯粹性。”
* **约束线程模型**：“本模块内部不私自派生（Spawn）任何后台工作线程，所有执行路径均寄生于 `[调用方/上位机]` 的线程上下文中运行，处于严格的同步阻塞调用模型。”

### B. 协议/结构体/内存布局 (Protocol & Layout)

* **结构体 ABI 约束**：“为彻底屏蔽跨 CPU 架构与编译器的 ABI 差异，本头文件中所有跨边界交互的结构体均强制应用 `#pragma pack(push, 1)`（1字节对齐）指令，并显式禁止在结构体中声明虚函数与指针成员，以保证内存布局的一致性。”
* **通信报文规约**：“上下位机交互报文采用固定帧头加变长负载的拓扑结构。多字节整型字段（如 `uint32_t`）在总线上传输时，严格遵循 `[小端序/Little-Endian]` 规约。网络传输层的分片与重组对本应用层完全透明。”
* **内存池布局策略**：“鉴于高频触发场景下的堆内存碎片化风险，系统启动阶段将在物理内存中预分配大小固定的连续大页内存（Large Pages）作为 DMA 缓冲区。运行时核心链路彻底杜绝动态 `malloc/new` 行为。”

### C. 生命周期与资源管理 (Lifecycle & Resource)

* **强制初始化约束**：“算法句柄的生成与销毁必须形成严格的生命周期闭环。调用方在触发 `Init` 接口时，必须显式传入合法的 `[工作 Mode 枚举值]`，以此为依据进行内部执行路径的物理隔离。本接口拒绝处理任何带有歧义的缺省/空入参，非法参数将直接抛出构建失败。”
* **状态流转单调性**：“系统内部状态机的流转遵循严格的单向单调性约束（`IDLE` -> `READY` -> `RUNNING`）。处于 `RUNNING` 阶段的上下文被置为只读（Read-only）锁定状态，期间任何配置变更指令均作静默丢弃处理。”
* **句柄卸载约束**：“在执行 `Destroy` 动作前，调用方有义务确保所有寄生于该句柄的异步测算任务已彻底收敛或被显式中断，否则将导致内部悬挂指针访问及严重的段错误。”

### D. 容错/异常/降级机制 (Fault-tolerance & Degradation)

* **输入防御性校验**：“面对非可信的外部网络/串口输入，协议解析层在此构建前置防御性校验矩阵，包含但不限于：越界帧长拦截、CRC 完整性校验与魔数（Magic Number）匹配。未能穿透校验矩阵的脏数据将被即刻丢弃，切断其向核心业务层传播的通路。”
* **超时熔断与退化**：“当 `[核心算法测算]` 链路的执行耗时突破预设的 `[100ms]` 竞争窗口边界时，监控看门狗将触发超时熔断机制。系统强制挂起当前工作上下文，执行状态机重置，并向上游抛出 `ERR_TIMEOUT` 异常，主动切入降级恢复路径以保障次轮触发的可用性。”
* **异常隔离边界**：“针对单个 `[通道/任务]` 内发生的断言失败或段违例，系统将利用独立的进程空间/线程局部存储（TLS）进行异常隔离，防止毒药负载引发全局服务的级联失效（Cascading Failure）。”
