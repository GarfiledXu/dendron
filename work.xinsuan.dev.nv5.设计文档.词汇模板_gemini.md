本文档旨在提供一套标准化的词汇、命名公式与句式模板，用于在编写工程设计文档时提升表达的严谨性、客观性与专业度，消除口语化与歧义。

## 1. 核心标题命名规范 (Naming Conventions)

高质量的工程段落标题应具备排他性，标准组合公式为：
**`[前缀限定语/作用域] + [核心实体/名词] + [属性描述/后缀]`**

### 1.1 核心后缀词汇池 (替换“说明/详解/处理”)

| 工程名词 (后缀) | 适用场景 / 核心暗示 | 组合标题示例 |
| :--- | :--- | :--- |
| **规范 (Specification)** | 强制执行的标准，不可违背的数据格式或代码标准。 | `跨平台 C++ 内存对齐规范`、`标准工程 src 目录结构规范` |
| **规约 (Convention)** | 涉及多方交互、需要共同遵守的边界契约。 | `上下位机串口通信报文规约`、`算法库接口交互规约` |
| **机制 (Mechanism)** | 维持某项功能运转的内在逻辑闭环（含触发、执行、反馈）。 | `异常中断恢复机制`、`文件句柄并发读写机制` |
| **策略 (Strategy/Policy)** | 在多种可行性中选择的一种倾向性方案（如重试、缓存）。 | `高频采集下的图像内存池调度策略`、`超时重试避让策略` |
| **流转 (Pipeline/Flow)** | 强调数据或控制信号在多个节点之间的先后加工顺序。 | `多通道图像采集与预处理流转路径`、`核心状态机流转` |
| **拓扑 (Topology)** | 描述物理或逻辑实体之间的网状/层级关系。 | `系统进程间通信拓扑`、`网络拓扑架构` |
| **生命周期 (Lifecycle)** | 描述对象从分配内存、初始化、工作到销毁的全过程。 | `算法句柄的生成、持有与释放生命周期` |
| **基准 (Benchmark)** | 性能或精度的底线与测量标准。 | `OCR 测算精度与执行耗时基准` |
| **模型 (Model)** | 对复杂业务逻辑或数据结构的抽象。 | `测算结果输出数据模型` |

### 1.2 提升严谨度的修饰定语 (前缀词汇池)

在核心名词前增加以下定语，可准确界定设计意图与系统级特性：

* **资源与内存层面**
  * **预分配 (Pre-allocated)**：强调资源提前就绪（例：`预分配内存池`）。
  * **零拷贝 (Zero-copy)**：强调避免数据搬运损耗（例：`图像数据零拷贝流转`）。
  * **隔离 (Isolation)**：强调上下文的安全边界（例：`多 Mode 初始化状态隔离`）。
  * **收敛 (Convergence)**：强调逻辑入口的统一（例：`底层硬件状态查询收敛`）。
* **时序与并发层面**
  * **同步 (Synchronous) / 异步 (Asynchronous)**：明确调用方的等待状态。
  * **阻塞 (Blocking) / 非阻塞 (Non-blocking)**：明确线程的执行状态。
  * **无锁 (Lock-free) / 线程安全 (Thread-safe)**：明确并发环境下的安全承诺。
* **系统健壮性层面**
  * **防御性 (Defensive)**：强调对非预期行为的防范（例：`防御性指针校验`）。
  * **幂等性 (Idempotent)**：强调重复执行的安全性（例：`配置指令下发幂等性`）。
  * **兜底 (Fallback) / 降级 (Degradation)**：强调异常情况下的次优退路（例：`执行超时兜底策略`）。
* **数据状态层面**
  * **有状态 (Stateful) / 无状态 (Stateless)**
  * **持久化 (Persistent) / 易失性 (Volatile)**

## 2. 万能句式模板库 (Paragraph Templates)

在撰写具体章节内容时，可采用以下标准句式作为段落的开篇，以确立客观严谨的基调。

### 2.1 概述与边界划定

* **阐述设计目的**：“本模块设计的核心目的在于剥离 `[上游模块]` 与 `[下游模块]` 的强耦合，通过引入 `[核心机制]`，实现 `[业务场景]` 下的高效流转。”
* **界定功能边界**：“本章节仅定义 `[某功能]` 的控制流转与状态机变更规范，不包含底层的具体业务逻辑运算实现。”

### 2.2 数据结构与接口约束

* **声明数据规范**：“为消除不同编译器/硬件平台间的解析歧义，本模块对所有交互结构体实行严格的 `[如: 4字节]` 内存对齐规范，并采用统一的 `[如: 小端序]` 字节序。”
* **强调初始化约束**：“系统要求上游调用方在创建句柄时，**必须**显式指定初始化 Mode 枚举值，以确保内部算法逻辑路径的正确路由与资源隔离，拒绝接受未定义的缺省状态。”

### 2.3 并发控制与资源调度

* **描述内存管理策略**：“鉴于工业级高频触发场景下的算力与时延约束，本机制禁止在业务执行链路中进行大块内存的动态分配（禁止运行时 `new/malloc`）。所有图像 Buffer 及输出结果结构体均采用预分配池化调度策略。”
* **描述生命周期闭环**：“资源由 `[申请方]` 负责分配与持有，当且仅当触发 `[特定条件/事件]` 时，交由 `[接收方]` 执行句柄的最终销毁，形成严格的生命周期闭环，避免内存泄漏。”

### 2.4 异常处理与防呆设计

* **描述前端拦截机制**：“针对总线报文残缺、长度越界或 CRC 校验失败的非法输入，下位机调度层将采取防御性拦截策略，直接丢弃该帧并向上游回抛标准的 `[如: ERR_CRC_MISMATCH]` 状态码，避免污染后续算法执行队列。”
* **描述超时兜底策略**：“若由于外部硬件中断或不可控因素导致核心执行链路耗时越界（超限 `[X]ms`），系统将强制触发异常兜底策略，主动挂起/终止当前测算任务，并重置内部状态机至 `IDLE` 状态。”
